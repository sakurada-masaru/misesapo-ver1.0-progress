# ビューポート単位（vw/vh）の説明

## ビューポート単位とは？

**ビューポート単位（vw/vh）は、実際のビューポート（ブラウザの表示領域）のサイズに基づいて計算される単位です。**

### 重要なポイント

❌ **誤解**: 「SP画面でも実際のビューポートはPC並に設定しておく」  
❌ **誤解**: 「表示される領域はスマホ画面サイズだが、実際には画面外にビューエリアが存在している状態」

✅ **正解**: 
- **ビューポート = 実際に画面に表示されている領域**
- スマホならスマホのビューポート（例: 375px × 667px）
- PCならPCのビューポート（例: 1920px × 1080px）
- **画面外にビューエリアが存在するわけではない**

---

## vw/vh の仕組み

### 基本単位
- **1vw = ビューポート幅の1%**
- **1vh = ビューポート高さの1%**

### 例：
```css
/* スマホ（375px幅）の場合 */
width: 50vw;  /* = 375px × 0.5 = 187.5px */

/* PC（1920px幅）の場合 */
width: 50vw;  /* = 1920px × 0.5 = 960px */
```

**同じCSSでも、画面サイズに応じて自動的にサイズが変わる！**

---

## より多くのカードを表示する方法

### 現在の実装（固定サイズ）
```css
.task-card {
    width: 280px;  /* 固定サイズ */
    height: 180px; /* 固定サイズ */
}
```

**問題**: 
- スマホでは1-2枚しか表示できない
- PCでも固定サイズのため、画面の大きさを活かせない

### ビューポート単位を使用した改善例

#### 方法1: カードサイズをビューポート単位で指定
```css
.task-card {
    /* 画面幅の30%（約3列表示） */
    width: calc(30vw - 20px);
    /* 画面高さの20%（約5行表示） */
    height: calc(20vh - 20px);
}

/* スマホ（375px幅 × 667px高さ）の場合 */
/* width = 375px × 0.3 - 20px = 92.5px */
/* height = 667px × 0.2 - 20px = 113.4px */

/* PC（1920px幅 × 1080px高さ）の場合 */
/* width = 1920px × 0.3 - 20px = 556px */
/* height = 1080px × 0.2 - 20px = 196px */
```

#### 方法2: グリッドレイアウトと組み合わせ
```css
.task-grid {
    display: grid;
    /* 画面幅に応じて自動的に列数を調整 */
    grid-template-columns: repeat(auto-fill, minmax(calc(30vw - 20px), 1fr));
    gap: 10px;
}

.task-card {
    width: 100%;  /* グリッドセルに合わせる */
    height: calc(20vh - 20px);
}
```

#### 方法3: メディアクエリで段階的に調整
```css
/* スマホ */
@media (max-width: 430px) {
    .task-card {
        width: calc(33.33vw - 12px);  /* 3列表示 */
        height: calc(25vh - 12px);    /* 約4行表示 */
    }
}

/* タブレット */
@media (min-width: 431px) and (max-width: 1024px) {
    .task-card {
        width: calc(25vw - 15px);  /* 4列表示 */
        height: calc(20vh - 15px); /* 約5行表示 */
    }
}

/* PC */
@media (min-width: 1025px) {
    .task-card {
        width: calc(20vw - 20px);  /* 5列表示 */
        height: calc(15vh - 20px);  /* 約6-7行表示 */
    }
}
```

## 実際の効果

### 現在の実装（固定サイズ）
- スマホ: 約1-2枚のカードが表示
- PC: 約6-8枚のカードが表示（画面の大きさを活かせない）

### ビューポート単位を使用した場合
- スマホ: 約9-12枚のカードが表示（3列 × 3-4行）
- PC: 約25-35枚のカードが表示（5列 × 5-7行）

**画面サイズに応じて、自動的に適切なサイズに調整される！**

---

## まとめ

1. **ビューポート単位（vw/vh）は実際のビューポート（表示領域）のサイズに基づく**
   - 画面外にビューエリアが存在するわけではない
   - スマホなら小さいビューポート、PCなら大きいビューポート

2. **より多くのカードを表示するには**
   - カードサイズをビューポート単位で指定
   - 画面が大きいほど、自動的にカードが小さくなる（相対的に）
   - 同じ画面サイズでも、カードを小さくすればより多くのカードを表示可能

3. **実装のポイント**
   - 固定サイズ（px）ではなく、ビューポート単位（vw/vh）を使用
   - メディアクエリでデバイスごとに最適なサイズを設定
   - グリッドレイアウトと組み合わせて柔軟な配置
